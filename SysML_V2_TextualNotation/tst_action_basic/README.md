## SysML v2 Action Example (`tst_action_basic`)

This example shows how a simple SysML v2 action model can be executed from C++ using code generated by the sinelabore tools.

### `test.sysml`

`test.sysml` demonstrates attributes, action definitions, parameters, local variables, action calls, and decision logic in SysML v2.

### `main.cpp`

`main.cpp` is a minimal driver that instantiates and runs the generated C++ code for the SysML model:
- **Creates** an instance of `Test::A`.
- **Initializes** it by calling `a.init()`.
- **Creates** an execution object `Test::A::runDef run` representing one execution of the SysML `run` action.
- **Sets the input** `run.n = 0` for the action.
- **Executes** the action by calling `run(a)`.
- **Prints the outputs** `run.p` and `run.u` (the results of the `run` action) to standard output.

This setup illustrates how a SysML v2 action definition (`run` in `test.sysml`) maps to generated C++ code.
In C++, each SysML action is represented by a generated struct (here `Test::A::runDef`) whose data members correspond to the action’s parameters and local variables. 
Inputs such as `n` and outputs such as `p` and `u` are accessed via fields like `run.n`, `run.p`, and `run.u`. 
You set the input fields before calling the struct as a function with the owning part instance, and after the call the output fields contain the computed results of the action.
